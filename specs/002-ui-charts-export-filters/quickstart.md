# Quickstart: Ручное тестирование UI — Charts, Export & Filters

**Feature**: 002-ui-charts-export-filters
**Prerequisite**: Сервис запущен (`uv run uvicorn src.main:app --reload`) или `docker-compose up`

---

## 0. Подготовка тестовых данных

```bash
# Убедиться, что есть источник данных и прогон анализа
# Открыть http://localhost:8000 и запустить анализ через UI
# Или через curl:
curl -s http://localhost:8000/api/v1/sources | python -m json.tool
curl -X POST http://localhost:8000/api/v1/analysis/run \
  -H "Content-Type: application/json" \
  -d '{"source_id": "<source_id>", "date_from": "2026-01-01", "date_to": "2026-03-31"}'
```

Сохранить `analysis_id` из ответа.

---

## 1. Тестирование легенды на тепловой карте (Phase B)

1. Открыть `http://localhost:8000/heatmap?run_id=<analysis_id>`
2. **Проверить**: Легенда отображается рядом с картой или под ней
3. **Проверить**: Легенда содержит все 6 типов аномалий с цветными метками и описаниями
4. **Проверить**: Легенда видна без прокрутки страницы (viewport 1280×768)
5. **Проверить**: При наведении на ячейку с аномалией показывается тултип с типом и значением
6. Уменьшить окно до 1280px — горизонтальная прокрутка не появляется

---

## 2. Тестирование Plotly.js на drilldown (Phase C)

1. Кликнуть на любую ячейку тепловой карты → откроется `/drilldown?...`
2. **Проверить**: График отображается (нет пустого контейнера, нет JS-ошибок в консоли)
3. **Проверить**: Нормальные точки — зелёные, аномальные — в цвете соответствующего типа
4. **Проверить**: Кнопки Plotly toolbar (zoom, pan, reset) работают
5. Навести курсор на аномальную точку → тултип показывает: Тип, Значення, Відхилення %
6. Навести на нормальную точку → тултип показывает: Період, Значення
7. **Проверить**: Легенда ряда справа от графика перечисляет типы
8. Изменить размер окна браузера → график перестраивается (responsive)

---

## 3. Тестирование расширенных фильтров на таблице (Phase E)

1. Открыть `http://localhost:8000/table?run_id=<analysis_id>`
2. **Мультивыбор типов**: Снять все галочки кроме `SPIKE` → таблица показывает только SPIKE
3. **Текстовый поиск**: Ввести часть имени в поле поиска → после 300ms таблица фильтруется
4. **Порог отклонения**: Ввести `50` в поле «Мін. відхилення %» → остаются строки с |pct_change| ≥ 50
5. **Комбинация**: Выбрать тип SPIKE + ввести текст + порог → все три фильтра работают вместе
6. **Сброс**: Нажать «Скинути фільтри» → все поля очищаются, таблица показывает все аномалии

---

## 4. Тестирование экспорта в Excel (Phase F)

1. На странице таблицы применить любой фильтр (например, только SPIKE)
2. Нажать «Експорт в Excel»
3. **Проверить**: Браузер скачивает файл `anomalies_<date>.xlsx`
4. Открыть файл в Excel/LibreOffice Calc
5. **Проверить**: Заголовки — кириллица: Період, Тип аномалії, Поточне значення, …
6. **Проверить**: Строки соответствуют отфильтрованным данным в таблице
7. Сбросить фильтры, снова экспортировать → в файле больше строк
8. Применить фильтр с нулевым результатом, экспортировать → файл содержит только заголовки

---
